# Package Version Monitor

Система для сбора и отслеживания версий программных пакетов на удалённых Linux-хостах с использованием Ansible.

Проект автоматически собирает версии указанных пакетов, сравнивает их с предыдущим состоянием и сохраняет историю изменений.

# Принцип работы

1. Скрипт `run.sh` запускает playbook `playbooks/collect.yml`.

2. Playbook копирует на удалённые машины скрипт `scripts/get_versions.sh` и выполняет его.

3. Скрипт собирает версии установленных пакетов и возвращает результат в формате JSON.

4. Полученные JSON-файлы сохраняются в каталог `data/incoming/`.

5. Скрипт `compare/compare.py`:
   - сравнивает новые данные с текущим состоянием (`data/current/`);
   - при обнаружении изменений обновляет текущее состояние;
   - записывает изменения в журнал (`data/history/`);
   - удаляет обработанный файл из `data/incoming/`.

# Подготовка окружения

На компьютере с Windows откройте PowerShell и выполните:

```powershell
wsl --install
```

После установки WSL и Ubuntu откройте терминал Ubuntu и выполните:

```bash
git clone https://github.com/ivonki/package-version-monitor.git
cd package-version-monitor
bash install.sh
```

Скрипт `install.sh` автоматически:

- обновляет список пакетов;
- устанавливает Ansible;
- устанавливает Python и необходимые модули;
- создаёт виртуальное окружение Python;
- устанавливает зависимости проекта;
- выдаёт права на выполнение скриптов.

# Настройка inventory

Заполните файл:

```text
inventory/hosts.ini
```

Пример:

```ini
[all]
server1 ansible_host=10.10.10.10 ansible_user=developer
server2 ansible_host=10.10.10.11 ansible_user=developer
```

где:

- `server1` — произвольное имя хоста;
- `ansible_host` — IP-адрес или DNS-имя удалённой машины;
- `ansible_user` — пользователь для подключения по SSH.

Файлы с результатами будут сохраняться под именем IP-адреса удалённой машины (`ansible_host`).

---

# Настройка SSH

Для работы Ansible необходимо настроить аутентификацию по SSH.

Если SSH-ключ отсутствует, создайте его:

```bash
ssh-keygen -t rsa
```

Нажимайте `Enter`, чтобы принять параметры по умолчанию.

Скопируйте открытый ключ на каждый удалённый хост:

```bash
ssh-copy-id developer@10.10.10.10
ssh-copy-id developer@10.10.10.11
```

где:

- `developer` — значение `ansible_user`;
- `10.10.10.10` — значение `ansible_host`.

Проверьте подключение:

```bash
ssh developer@10.10.10.10
```

Если вход выполняется без запроса пароля, настройка завершена успешно.

# Автоматический запуск

Для автоматического мониторинга рекомендуется использовать `cron`.

Откройте редактор:

```bash
crontab -e
```

Добавьте задачу, например для запуска каждые 10 минут:

```cron
*/10 * * * * /home/<user>/package-version-monitor/run.sh >> /home/<user>/package-version-monitor/logs/cron.log 2>&1
```

При необходимости периодичность можно изменить.

# Требования

## Управляющая машина

- Windows 10/11 с WSL Ubuntu;
- подключение к сети Интернет;
- права `sudo`.

Все необходимые зависимости устанавливаются автоматически скриптом `install.sh`.

## Удалённые машины

- Linux;
- SSH-сервер;
- Bash;
- наличие отслеживаемых программ (php, php-fpm, apache, nginx, openssl, python, npm и др.).

Python на удалённых машинах не требуется.

---

# Особенности

- не требует установки агента на удалённых машинах;
- использует SSH и Ansible для выполнения команд;
- автоматически отслеживает изменения версий пакетов;
- хранит историю изменений;
- поддерживает запуск по расписанию с помощью `cron`.
